Модель монетизации MCP

MCP НЕ монетизируется напрямую. Рынок ожидает бесплатный MCP (Stripe, Dice, Himalayas — все бесплатны). Модель: бесплатный MCP привлекает трафик → конвертирует в платную подписку. Для работодателей: бесплатный MCP с лимитами (5 запросов/день) → платный с безлимитом. Для соискателей: бесплатный submit_resume через MCP → upsell в карьерный клуб.

Пошаговая GEO-стратегия для СБОРКИ

1

Schema.org JobPosting на каждую вакансию

Структурированные данные — главный фактор для LLM. JSON-LD разметка: title, salary, skills, company, location. Google и AI-поисковики парсят это первым.

2

Answer-first контент в 150 статьях

Первый абзац каждой статьи = прямой ответ на запрос. LLM извлекают первый чёткий ответ. Переструктурировать все 150 статей: таблицы зарплат, bullet-списки, FAQ-секции.

3

10 "зарплатных страниц" — максимальный LLM-бейт

"Средняя зарплата [роль] в России 2026" — самый частый запрос к AI. Создать 10 definitive pages с реальными данными HH API. LLM будут цитировать как авторитетный источник.

4

MCP как прямой канал в AI

Когда пользователь спрашивает Claude "найди мне вакансию Python в Москве" — Claude может вызвать MCP СБОРКИ напрямую и вернуть реальные вакансии. Это НЕ SEO — это прямой API-канал в ответы AI.

5

FAQ на каждой странице

FAQPage Schema + реальные вопросы. LLM обожают FAQ — это формат Q&A, идеально маппится на промпт пользователя. У нас уже 13 FAQ на sborka.work, нужно 50+.

6

Регистрация MCP на всех директориях

Smithery (7 300+ tools), Apify (130K+ users/мес), mcpservers.org, MCP.so, PulseMCP, Glama. Каждая директория = backlink + discoverability для AI-агентов. Уже зарегистрированы на 4 из 6.

Позиционирование

СБОРКА — это НЕ конкурент HH.ru по контенту. Это AI-native карьерная платформа, оптимизированная для поиска работы через искусственный интеллект. Когда человек спрашивает ChatGPT о работе в России — СБОРКА появляется в ответе (GEO). Когда AI-агент компании ищет кандидатов — он вызывает MCP СБОРКИ (API). HH.ru не оптимизирован ни под одно, ни под другое.

06 / Антипаттерны

Что НЕ делать

Самые частые ошибки при построении job-маркетплейса. Каждый пункт — реальный кейс провала.

✕ Не брать деньги за MCP

Рынок ожидает бесплатный MCP. Stripe, Dice, Indeed — все бесплатны. Платный MCP убьёт adoption на старте.

✕ Не конкурировать по объёму

HH.ru = 70% рынка, 2.8M вакансий. Невозможно догнать. Конкурировать по качеству matching и AI-доступности.

✕ Не строить свой TTS/ML

Mercor при $450M ARR использует внешние LLM. Не инвестировать в ML-инфраструктуру — использовать API (OpenAI, Groq, DeepSeek).

✕ Не ждать юзеров для GEO

AI-трафик приходит от LLM, а не от юзеров. Оптимизируйте контент сейчас — AI-цитирование растёт независимо от вашей базы.

✕ Не делать "ещё одну джоб-борду"

Moonhub ($3.8M ARR) не выжил как standalone — продан Salesforce. Позиционирование: AI-native, не "ещё одна борда".

✕ Не запускать без контента

Пустой маркетплейс = chicken-and-egg. Сначала агрегировать вакансии с HH/SuperJob, потом привлекать прямых работодателей.
